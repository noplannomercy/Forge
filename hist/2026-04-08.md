## Cortex ← Forge 연동 리뷰 (2026-04-08)

### 연동 가능 상태

| Cortex 쪽 | Forge 쪽 | 상태 |
|-----------|---------|------|
| `/v1/ingest/file` → 외부 chunker 호출 패턴 (`CHUNKER_URL`) | `/convert` API | 패턴 동일, 라우팅만 추가 |
| LLM 메타 추출 (`metadata.py` — 5개 키) | MetaExtractor (category, title 등) | **중복 — 결정 필요** |
| PostgreSQL (`cortex_db`) | `forge_` 테이블 (같은 DB) | 연결됨 |
| API key 인증 (`X-API-Key`) | `X-Forge-Key` | 패턴 동일 |

### 결정 필요 항목

1. **메타 추출 주체** — Cortex도 LLM으로 메타 뽑고 Forge도 뽑음. 중복. Forge가 뽑고 Cortex에 넘기는 게 맞을 듯 (변환 직후가 타이밍 최적)
2. **비동기 호출 방식** — Forge는 비동기 Job (poll). Cortex `ingest/file`에서 poll 루프 필요하거나, Forge에 `?sync=true` 또는 `callback_url` 추가
3. **Redis 타이밍** — Cortex Phase 3 Redis 도입 시 Forge도 같이 전환하면 인프라 공유
4. **chunking 역할** — Forge는 마크다운 반환만, 청킹은 Cortex (역할 분리 깔끔)

### Cortex 연동 포인트

```
cortex/core/chunker.py  ← Forge /convert 호출
cortex/core/ingest.py   ← 변환 결과 받아서 기존 파이프라인
cortex/config.py        ← FORGE_URL 환경변수 추가
```
