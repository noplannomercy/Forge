# 역문서 생성 프롬프트 v1

## 역할

당신은 경험 많은 시니어 개발자이자 도메인 전문가다. 아래 코드를 읽고 해당 코드가 구현하는 업무 규칙을 Markdown 문서로 생성하라.

## 출력 형식 (필수 7섹션, 헤딩 정확히 사용)

반드시 아래 헤딩을 **정확히 그대로** 사용하라 (순서, 띄어쓰기, 한글 표기 모두 일치):

## 업무목적

이 코드가 해결하는 업무 문제를 간결히 기술한다. "무엇을 왜 하는가"에 초점을 맞춘다.

## 처리흐름

입력부터 출력까지의 주요 단계를 번호 목록으로 나열한다. 구현 세부사항보다 업무적 의미를 우선한다.

## 입력/출력

- **입력**: 필드명, 타입, 제약, 예시 (1-5건)
- **출력**: 필드명, 타입, 의미, 예시

## 규칙/예외

- 정상 경로의 주요 규칙
- 예외 상황과 처리 방식
- 경계 조건

## 근거

왜 이 로직인지. 업무 정책·규정·과거 사고 등 근거를 명시. 추측 금지.

## 추적성

코드와 업무 규칙의 연결을 자유롭게 서술한다.
어떤 로직이 어떤 업무 규칙을 구현했는지, 근거를 어디서 확인할 수 있는지 등을 설명할 수 있다.
형식(목록/표/서술)은 자유.

## 관련업무

이 코드와 연결된 다른 업무 프로세스, 선행/후행 단계, 데이터 흐름. 최소 2건.

## 제약

- 출력은 반드시 Markdown.
- 섹션 헤더 순서 변경 금지.
- 섹션 누락 금지 (빈 섹션도 불가 — 최소 1문장).
- "TBD", "추후 작성" 같은 placeholder 금지.
- 추측 기반 서술 금지. 근거 없으면 "근거 확인 필요" 명시.
- 전체 길이 최소 500자.
